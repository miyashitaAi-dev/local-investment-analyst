## F. 今後の再検証で追加すべき分類軸（不具合15の知見を反映）

| 難度軸 | 易（低リスク） | 難（高リスク） |
|---|---|---|
| 通期予想の修正頻度 | 期中修正なし、または1回のみ（トヨタ） | 期中に複数回修正（ソニー：3回） |
| 事業区分の変更 | 通期で区分変更なし | 期中に事業の分離・統合・売却目的保有への分類変更あり（ソニー：金融事業分離、トヨタ：日野自動車の連結除外） |

## D. 総合判定

- [x] A（正確性）：直近3銘柄で一致率◯割以上 → 直近期の主要指標（現在値・PER・PBR等）は概ね正確。MACDは一貫して不正確
- [x] B（再現性）：3回実行して数値が安定 → 主要指標は安定。決算表重複が約33%の確率で発生
- [x] C（多様性）：4パターン全てで型崩れ・創作が起きない → 概ね良好。事業部門利益率・実績予想の区別に残存リスク

3つ全てクリアできたら「実用フェーズ」へ移行する目安とする。1つでも不安定なら、Skill・コアプロンプートの該当部分を見直す。

**最終結論（2026年6月26日）：** プロンプトエンジニアリングによる改善はほぼ尽くした。残存リスク（決算表重複・事業部門利益率/実績予想の区別・MACD精度）は、これ以上の発生率ゼロ化をプロンプトだけで目指すのではなく、①人間による最終確認を運用ルールとする、②ファインチューニングの教師データで重点的に改善する、という次フェーズに進む。
